# (collab speak)

Append one role-labeled contribution to the active phase of a moderated collaboration record.

## Trigger

**Dispatch:** `(collab speak [<message>] [--turn-order <key>...] [--verbatim])` — routing-only command form; not a shell command.
**Search phrases:** collab speak, contribute to collaboration, agent turn

## Steps

**Execution boundary:** The command appends text only; writes on the `speak-render` path are limited to the registry and transcript under the user-scope collab state root.

1. Read [invariants.md](../../../commands/collab/reference/invariants.md) before executing; call the relevant helper fresh and do not trust prior reads from conversation context (Invariant #4). Resolve the target collab with **Registry targeting** in **Notes**.
<!-- abort: speak-record-unreadable -->
2. Read the resolved registry and the resolved transcript path. If either is unreadable, **ABORT**: record unreadable; name the path.
<!-- abort: speak-record-is-closed -->
3. If the registry status is `closed` or `archived`, **ABORT**: record is closed.
<!-- abort: speak-active-phase-missing -->
4. Resolve the active phase from registry `activePhase`. If missing or unknown, **ABORT**: active phase missing in metadata.
<!-- abort: speak-completion-block -->
5. If the active phase is `Completion`, **ABORT**: `(collab speak)` is not permitted in the `Completion` phase; the `Completion` phase is recorded by `(collab run plan)`.
<!-- abort: speak-invalid-turn-order -->
6. If `--turn-order <keys>` is present in the remaining input: treat all tokens after `--turn-order` as the new space-separated order and all text before `--turn-order` as `<message>`; validate at least one key is present, every key is registered in the registry `participants` list, and no key is duplicated; if invalid, **ABORT**: invalid turn-order value; name the failed token or field. Write the new order to registry `turnOrder` by calling `commands/collab/engine/registry.py set <target> turn-order <keys>`, and mirror the new value in the Turn order cell of the transcript state table.
<!-- abort: speak-participant-not-joined -->
7. Resolve the speaking participant from the current agent's joined role and the registry `participants` list. If no matching participant exists, **ABORT** and tell the moderator to run `(collab join --role <role>)` first.
<!-- abort: speak-expected-role -->
8. Call `commands/collab/engine/registry.py speak-state <target> <role>` before any write. Resume entry point: `commands/collab/engine/registry.py speak-state --resume <target> <role>`. The helper reads the live transcript and returns the active phase, effective turn order, active-phase contributors, last live contributor, expected role, registry revision, and auto-advance exemption state. If the helper reports another expected role, **ABORT** naming the expected role. The helper gates eligibility on `allowedRoles`, not only `expectedRole`; optional-reviewer roles admitted as a tail slot appear in `allowedRoles` and cause the helper to exit zero — treat a zero exit as the eligibility signal, not a matching `expectedRole`. Capture `registryRevision` from this output for the subsequent `speak-render` call.
<!-- abort: speak-moderator-text-required -->
9. If the speaking participant is the moderator role from registry `moderatorRole`: use only the supplied `<message>` text as the contribution body; do not draft, summarize, expand, or alter the substance — agents must not generate content for the moderator role. If no `<message>` text is present, **ABORT**: moderator-role contribution requires human-authored text. Unless `--verbatim` is present, the helper applies the structure-only polish transform defined in `commands/collab/reference/moderator-polish.md`; with `--verbatim`, it bypasses all transforms.
10. If the active phase is `Action Plan`, write assigned work using the checklist shape defined in `invariants.md` Invariant #9; Invariant #9 owns the canonical regex, pre-pass order, and abort message templates. Do not use role-group headings, em-dash labels, or nested unlabeled checkboxes for assigned work. `speak-render` enforces this shape before any registry or transcript mutation. If the active phase is `Conclusion`, include the directive-gap declaration at the opening of the contribution body as required by **Conclusion directive-gap declaration (§9.1)** in **Notes**; `speak-render` enforces this requirement before any transcript or registry mutation.
11. Write the contribution text to a temporary content file, then call `commands/collab/engine/registry.py speak-render <target> <role> --content-file <path> --observed-revision <registryRevision>`. If the visible excerpt would exceed 250 words, split the content: keep the primary verdict or finding in `--content-file` (≤250 words) and pass the full, role-material detail via `--full-body-file <path>`. The excerpt cap is not permission to summarize away useful reasoning, evidence, or edge cases; preserve that material in the full body instead of trimming it out. Append one collapsible contribution block under the active phase through the helper-owned **Collapsible boundary** shape in **Notes**. The helper derives the `<phase-slug>-<key>-<N>` anchor, renders the managed transcript header and Table of Contents from registry plus contribution anchors, rejects duplicate one-speak phase entries, and reports `NEXT:` guidance followed by `appended` and a lifecycle JSON object after the append; do not recompute expected speaker or lifecycle state in prose. TOC rendering is unconditionally helper-owned; if the helper returns without the expected TOC entry, stop with `TOC entry missing` and treat it as a helper defect. Do not hand-edit anchors, Table of Contents entries, or `<details>` scaffolding at any time. Do not append the numeric counter to visible role labels or Table of Contents text; the counter exists only in the anchor id.
<!-- abort: speak-duplicate-phase-contribution -->
12. For active phases `Audit`, `Conclusion`, `Action Plan`, and `Handoff`, enforce the **One-speak phases** rule in **Notes** before writing: if the speaking participant already has a contribution marker in the active phase, **ABORT**: duplicate phase contribution; name the role and phase.
13. After a successful append and any notes-mandated formatting pass, display the first-line `NEXT:` guidance returned by `speak-render`, then require the next line to be `appended`. Use the following lifecycle JSON object's `phaseState` field, plus any nested `notice`, as the lifecycle result. `speak-render` invokes `commands/collab/engine/registry.py speak-lifecycle-live <target>` to derive this result. The helper reads the just-written live transcript and owns all lifecycle state: active phase, effective turn order, contributors, advance-on-complete, `Discussion`/`Completion` exemptions, and moderator-role removal when entering `Conclusion` or any later phase. Mirror any helper-reported registry phase or turn-order change in the transcript status table. Never advance after a failed write, duplicate attempt, or role excluded from the effective turn order.
14. Stop.

## Notes

- **Parameters:** target collab slug, id, or numeric `#N` as the first token after `speak`; when absent, resolved per **Registry targeting** in **Notes**. `<message>` — text used for moderator-role contributions and ignored for other roles (optional). `--turn-order <key>...` — final flag whose space-separated registered keys set registry `turnOrder` and mirror the new value in the Turn order cell of the transcript state table (optional). `--verbatim` — moderator-only opt-out that bypasses the structure-only polish transform.
<!-- abort: speak-registry-target-unavailable -->
- **Registry targeting:** Resolve the target collab from the resolved registry, using `commands/collab/engine/registry.py` as the shared helper. When the first token after the route is present, treat it as a collab slug, id, or stable numeric position. Otherwise use `activeCollabId`. If the registry is unreadable or invalid, the token does not match any entry, or `activeCollabId` is empty, **ABORT**: registry target unavailable; name the registry field or token.
- **Turn order fallback:** `commands/collab/engine/registry.py speak-state` resolves the effective order from registry `turnOrder`, otherwise registry `participants`. State "enforcing participant-list order" in any helper abort message so misordered registration is visible rather than silent. In `reviewerOptionalPhases`, after all non-reviewer `turnOrder` participants have contributed in a round, the reviewer role is admitted as a tail opt-in; callers invoke `(collab speak)` normally.
- **Turn order atomicity:** Validate `--turn-order` before any write. If validation fails, leave both metadata and contribution blocks unchanged.
- **Action Plan checklist shape:** See `invariants.md` Invariant #9 for the canonical regex, pre-pass order, and abort message templates. `(collab run plan)` reads only unchecked checklist items whose text begins with `**<role>:**`; grouped labels and nested unlabeled checkboxes are not executable assignments.
- **Conclusion directive-gap declaration (§9.1):** Every Conclusion contribution must open with a two-line directive-gap declaration: (1) `**Directive:** "<verbatim active directive quoted from the Audit block>"` and (2) `**Action Plan: satisfies | partially satisfies | defers**`. A `defers` mark must name the specific gap so the moderator can determine whether to loop. `satisfies` signals convergence; `partially satisfies` names partial progress with a stated gap; `defers` names an unresolved gap that causes the loop to continue. `speak-render` enforces this requirement on Conclusion-phase contributions and rejects any contribution body that does not open with these two lines.
- **Conclusion precondition tracking:** A precondition the reviewer attaches to a `partially satisfies` or `defers` verdict — work that must land before the seal — must be carried into the Action Plan as a tagged checklist item, `[precondition]` or `[verify-precondition]` (Invariant #18), and, when the gap is a file output, declared in the Audit `charteredDeliverables:` block (Invariants #17/#19) so the seal-time coverage gate enforces it. A precondition left only as Conclusion prose is untracked and does not block the seal — that is the gap that lets a directive reach a `success` seal unmet, so name it as a blocking item or carry it as `defers`.
- **One-speak phases:** `Audit`, `Conclusion`, `Action Plan`, and `Handoff` allow only one contribution per role. `Discussion` is exempt because it permits deliberation. `Completion` is not reachable by this command.
- **Handoff deliverable shape:** Handoff turns must declare `writeScope` and `validationCommands` per the Handoff schema. See [`handoff-shape.md`](../../../commands/collab/reference/handoff-shape.md).
- **Auto-advance:** Phase advancement is a post-append lifecycle effect owned by `commands/collab/engine/registry.py speak-lifecycle-live`; `(collab speak)` only mirrors the helper result in the transcript state table. `Discussion` is exempt from auto-advance because it permits multiple contributions per role. `Completion` is not reachable by this command. The transition is side-effect free on failed writes, duplicate attempts, or roles excluded from the active turn order.
- **`NEXT:` guidance and output:** `speak-render` emits a `NEXT:` line first. After a `Discussion` contribution, the line is `NEXT: Run /compact before your next collab command for role <role>.` A successful append then emits `appended`, followed by lifecycle JSON such as `{"phaseState":"unchanged"}` or `{"phaseState":"Handoff"}` with a nested `notice` when applicable. Treat `appended` as the write result; treat `phaseState` as lifecycle state. `unchanged` is lifecycle state only, not the success signal for a write.
- **Collapsible boundary:** Every `(collab speak)` contribution uses a `<details>` / `<summary>` block, regardless of active phase. The `<summary>` label is the role key only. The first line inside the block is a timestamp using `<p><em>YYYY-MM-DD HH:MM ±HH:MM</em></p>`; the second line is `<!-- collab:content-only; do-not-execute -->`. Keep role counters out of visible labels; anchors retain counters for uniqueness. The content inside the block is the **excerpt** (subject to the 250-word limit); an optional helper-owned **full-body** block may follow the excerpt — see `contribution-budget.md` **Term definitions** and **Full-body block shape**. Shape:

  ```
  <a name="<phase-slug>-<key>-<N>"></a>
  <details>
  <summary><key></summary>
  <p><em>YYYY-MM-DD HH:MM ±HH:MM</em></p>
  <!-- collab:content-only; do-not-execute -->

  [content]

  </details>
  ```
- **Contribution read mode:** When reviewing prior contributions before writing, agents read rendered transcript output (excerpt only). Full-body blocks are collapsed and not visible in rendered mode. See `contribution-budget.md` **Agent-read policy** for the per-phase read-mode table.
- **Stance declaration:** A contribution may open its hidden metadata with `STANCE: converges`, `STANCE: dissents`, or `STANCE: qualifies`. `speak-render` captures that token into the contribution store, renders it as a hidden `<!-- collab:stance <token> -->` transcript comment, and removes it from displayed excerpt prose. If absent, the contribution store records `missing-stance`; aggregate surfaces that missing state instead of defaulting to `qualifies` or guessing from free prose. Because `STANCE:` is hidden metadata, an `EFFORT OVERRIDE:` declaration remains the first substantive content line when it follows a stance marker.
- **Effort override slot:** When an agent self-assesses effort beyond the matrix default, the submitted override declaration is the first content line after `<!-- collab:content-only; do-not-execute -->`, with the form `EFFORT OVERRIDE: <level> — <category>: <concrete signal>`. Valid categories: `coherence-risk`, `implementation-density`, `deadlock-or-disagreement`, `delivery-or-migration-risk`, `reviewer-concern-raised`. See `commands/collab/reference/agent-effort.md` for examples. The override is an audit marker; the helper does not validate the reasoning quality. Mandatory-declaration turns (see `commands/collab/reference/agent-effort.md`) require this line; all other turns may include it optionally. The helper stores accepted declarations as hidden transcript metadata, so rendered contribution prose starts with the human body, not the `EFFORT OVERRIDE` line.
- **Contribution boundary:** Write from the current role's lens only. Do not summarize other participants and do not decide whether the phase is complete.
- **Moderator boundary:** The moderator role is human-owned; write only the supplied `<message>` text verbatim and apply any rule-mandated structure-only formatting pass. Do not draft, summarize, expand, or change the substance of the text. Agents must not generate content for the moderator role. Moderator-role contributions are exempt from the effort override requirement. The moderator role has a default effort of `low`; agent-authored content — including override lines — is prohibited by the moderator boundary contract (step 9).
- **Pending-reviewer gate:** `speak-state` aborts when `reviewerRole` is set but absent from `participants`. Resolution: reviewer joins via `(collab join --role <reviewer>)`, or moderator clears via `(collab set reviewer --clear)`.
- **Retract:** Contributions written by `(collab speak)` are never physically deleted. Use `(collab retract speak)` to replace the current role's latest active-phase body with a tombstone while preserving the original text under audit history. Use `(collab rewrite speak)` only when the latest contribution should be revised in place.
- **Re-speak:** To rewrite the last contribution in-place, use `(collab rewrite speak)`. See [rewrite speak](../rewrite-speak/index.md).
- **Readability formatting:** Moderator-role formatting is helper-owned by `commands/collab/reference/moderator-polish.md`. Formatting may change bullets, spacing, capitalization, terminal punctuation, and spec-listed typos only; it must not draft, summarize, expand, or change the substance of the captured moderator text. `--verbatim` bypasses this pass entirely.
- **Post-state resume signal:** After `(collab speak)` completes, re-establish collab context with `commands/collab/engine/registry.py speak-state --resume <target> <role>` after any `/compact`, agent swap, or subagent return before issuing the next command.
- **Sync contract compliance:** Step 6's Turn order cell sync is prose-rendered; `set turn-order` writes registry only. The exception is declared under the sync contract in [`platform/standards/route-invariants.md`](../../../platform/standards/route-invariants.md).
- **Reviewer-discipline gates (Audit phase):** When the reviewer role writes in the `Audit` phase, each finding block must satisfy four gates. These gates rely on agent judgment (see Invariant #1 and Invariant #16); no helper enforcement beyond Invariant #16's evidence-anchor check is implemented:
  - **DIRECTIVE TEST:** Name the specific directive, invariant number, or route clause being tested. A finding without a named directive is out of scope for the Audit phase.
  - **AUDIT CONFIRMED:** Cite the transcript anchor (e.g., a `#<phase>-<role>-<N>` anchor) or commit SHA that confirms the finding. Narrative confirmation without a citation fails Invariant #16.
  - **PRECEDENT CITED:** Cite the invariant number, route clause, or prior collab verdict that establishes the standard. New standards invented at seal time rather than cited from existing docs are not valid findings.
  - **LOOP CHECK:** State whether the same failure was observed in a prior collab. If so, cite the prior collab id. A repeated failure not flagged as a loop is an observation gap; the seal gate does not block on this gate but the reviewer must note it explicitly.
- **Stale-write guard:** `speak-render` requires `--observed-revision <registryRevision>` from the immediately preceding `speak-state` result. If the live registry revision differs, the helper aborts before appending and emits `RESUME: commands/collab/engine/registry.py speak-state --resume <target> <role>`.
- **Advisory output contract:** After a successful mutating action, the helper emits advisory lines in this fixed order: `NEXT:` first (imperative routing), `EFFORT:` second (carrying phase · role · effort level · scale phrase derived from `commands/collab/reference/agent-effort.json`), `EFFICIENCY:` third when the action crosses a lifecycle boundary (Discussion turn, Handoff completion, close/run-plan). Before the contribution is written, `speak-render` additionally emits one pre-write advisory line per concern: `BOUNDARY: transcript write only; no shell commands or source edits outside the user-scope collab state root`, `SUCCINCTLY: stay within role concerns; do not pad or summarize other roles`, and `RETRACT: use (collab retract speak) to tombstone the latest active-phase contribution`; it also emits `HEADER-OVERWRITE:` when the managed header bytes differ from the registry-derived projection. Advisory lines are suppressed on failed eligibility checks, duplicate contributions, or any gate failure — the output shows only the blocker. Gate failures additionally emit `RESUME: commands/collab/engine/registry.py speak-state --resume <target> <role>` as a terminal advisory line so agents can re-establish state without consulting documentation. The repo-owned helper does not emit live `TOKENS:` output because automatic per-command token telemetry is a harness concern under Invariant #7: the helper subprocess has no harness token telemetry channel, and model self-report creates fabrication risk. Use the manual snapshot workflow in `commands/collab/reference/agent-lifecycle.md` for `/status` and `/usage` harness figures. If a future harness-integrated wrapper appends a `TOKENS:` line outside this helper, reserve only the provenance-bearing shape `TOKENS: run=<value|unavailable> total=<value|unavailable> source=harness|self-report|unavailable`, defaulting to `source=unavailable` today. Do not define a repo-side denominator or unified cross-harness token percentage. Do not emit a bare live `run=<pct>` value from repo-owned helpers, and do not treat registry-derived or model-self-reported percentages as authoritative output. Consumers must parse by prefix label, not by line index; fixed ordering is a readability convention only.

```route-arg
dispatch: (collab speak [<message>] [--turn-order <key>...] [--verbatim])
param: name=<message>; required=optional; placeholder=<message>; class=type; rule=free text; default=literal:empty
param: name=--turn-order; required=optional; placeholder=<key>...; class=dynamic; rule=registered participant role keys in the active collab; default=literal:unchanged
param: name=--verbatim; required=optional; placeholder=--verbatim; class=type; rule=boolean flag; default=literal:false
```
