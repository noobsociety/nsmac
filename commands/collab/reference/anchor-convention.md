# Anchor convention

Defines the `<phase-slug>-<role>-<counter>` anchor shape used in collab transcripts. The convention is helper-enforced; the emitter is the single source of truth.

## Steps

2. When changing the anchor shape or emission logic, change both this document **and** the emitter (`commands/collab/engine/transcript_render.py`) in the same commit. A doc change without an emitter change, or vice versa, is a drift defect.
3. Do not mutate registry or transcript state from this documentation-only reference.

## Notes

**Anchor shape**

Every contribution block in a collab transcript carries an HTML anchor immediately above the `<details>` block:

```
<a name="{phase-slug}-{role}-{counter}"></a>
```

| Component | Value |
|---|---|
| `phase-slug` | Phase name lowercased with spaces replaced by hyphens (e.g., `action-plan`, `handoff`) |
| `role` | Participant role key (e.g., `tw`, `pe`, `pa`, `mod`) |
| `counter` | 1-based integer; increments per role per phase within the collab |

Examples: `audit-<role>-1`, `discussion-<role>-2`, `action-plan-<role>-1`, `handoff-<role>-1`.

**Uniqueness scope**

Anchors are unique within a collab. The counter ensures uniqueness when the same role contributes more than once in the same phase (e.g., `discussion-<role>-1`, `discussion-<role>-2`).

**Emitter**

The anchor is generated by `commands/collab/engine/transcript_render.py` at the `render_contribution_block` function:

```python
anchor = f'{phase_slug(phase)}-{role}-{counter}'
```

`phase_slug` is defined in `commands/collab/engine/normalizers.py`: `phase.lower().replace(' ', '-')`.

Counter assignment is computed by `next_anchor_counter` in `commands/collab/engine/registry.py`, which scans the live transcript for existing anchors with the same phase/role prefix and returns the next highest value.

**Maintainer note**

The convention is helper-enforced, not advisory. Changes to the anchor shape must be paired with changes to `transcript_render.py` and `normalizers.py`. A doc change that does not update the emitter, or an emitter change that does not update this doc, creates a spec/helper divergence (show-policy drift).

**References in the transcript**

Contributions are cross-referenced by anchor in Discussion, Conclusion, and Handoff prose using the form `#<phase-slug>-<role>-N`. Reviewer findings must cite at least one anchor or committed path per claim (Invariant #16, `commands/collab/reference/invariants.md`).
